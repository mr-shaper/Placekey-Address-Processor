支持包含地址信息的CSV文件，最大16MB

CSV文件格式要求
必需列（至少包含一列）：
  • address - 完整地址
  • 示例: "123 Main St Apt 4B, New York, NY 10001"
  • street_address - 街道地址
  • 示例: "123 Main St Apt 4B"
可选列：
  • city - 城市
  • 示例: "New York"
  • state - 州/省
  • 示例: "NY" 或 "New York"
  • postal_code - 邮编
  • 示例: "10001"
地址格式支持
  • 标准格式："123 Main St Apt 4B"
  • 波浪线分隔："123 Main St~~~Apt 4B" (系统会自动识别并处理)
  • 其他分隔符：支持 #、UNIT、APT、STE 等常见公寓标识
支持中英文列名，系统会自动识别。如果您的列名不同，上传后可以进行映射配置。

输出文件字段说明
原始数据字段
  • 地址/street_address: 原始地址信息
  • 收件人城市: 城市信息
  • 收件人省/州: 州/省信息
  • 收件人邮编: 邮政编码
  • 收件人国家: 国家信息
原规则分析
  • 是否公寓_原规则: 原规则判断结果
  • 置信度_原规则: 原规则置信度(0-100)
  • 匹配关键词_原规则: 匹配的关键词
Placekey API字段
  • placekey: 地址唯一标识符
  • placekey_confidence: API匹配置信度
  • placekey_matched_address: 标准化地址
  • placekey_success: API调用是否成功
  • placekey_error: API错误信息
  • placekey_latitude: 纬度坐标
  • placekey_longitude: 经度坐标
  • placekey_address_components: 地址组件
Placekey反向映射字段
  • reverse_mapping_success: 反向映射是否成功
  • reverse_mapping_address: 反向映射获得的完整地址
  • reverse_mapping_coordinates: 反向映射坐标信息
  • reverse_mapping_latitude: 反向映射纬度
  • reverse_mapping_longitude: 反向映射经度
  • reverse_mapping_error: 反向映射错误信息
增强分析字段
  • 公寓类型_增强: 识别的公寓类型
  • 主地址_增强: 提取的主地址
  • 公寓信息_增强: 详细公寓信息
  • unit_number: 提取的门牌号信息
整合结果
  • 是否公寓_整合: 最终判断结果
  • 置信度_整合: 最终置信度
  • 匹配关键词_整合: 整合关键词
  • 处理状态: 处理状态信息
  • 冲突标记: 是否存在结果冲突
重要说明：
  • Placekey相关字段需要有效的API密钥才能填充数据
  • 反向映射功能通过Placekey获取更准确的地址信息和坐标
  • unit_number字段提取的是地址中的公寓单元信息
  • 整合结果综合了原规则和API分析的最佳判断
  • 反向映射仅在成功获取Placekey后执行，提供额外的地址验证
